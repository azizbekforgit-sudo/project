# План: Полная система доставки — от выбора до.contact

## Общая логика

```
Покупатель                          Драйвер
    |                                   |
    |--- Выбирает "Внешняя доставка"    |
    |--- Выбирает маршрут (откуда/куда) |
    |--- Видит список драйверов         |
    |--- Кликает на драйвера            |
    |--- Видит профиль + калькулятор    |
    |--- Disclaimers (я прочитал/согласен)| 
    |--- Подтверждает выбор             |
    |                                   |
    |   ┌─── Заказ создан (pending) ───┐|
    |   │  Мои заказы: ожидание       │|
    |   │                              │|
    |   │  У драйвера: входящий заказ  │|
    |   │  Инфо: откуда/куда/груз     │|
    |   │  Disclaimers (я прочитал/согласен)
    |   │  Драйвер принимает           │
    |   └──────────────────────────────┘|
    |                                   |
    |--- Оба получают: "Свяжитесь по   |
    |    телефону: +998..."             |
    |    (через мини-чатбот)            |
```

### 1.3 Новая модель: `DeliveryRequest`

Связывает рыночный заказ с драйвером и хранит весь lifecycle:

**`models.py`** — новая модель:
```python
class DeliveryRequest(Base):
    __tablename__ = "delivery_requests"
    
    id              = Column(Integer, primary_key=True)
    order_id        = Column(Integer, ForeignKey("orders.id"), nullable=False)
    courier_id      = Column(Integer, ForeignKey("users.id"), nullable=False)
    buyer_id        = Column(Integer, ForeignKey("users.id"), nullable=False)
    
    route_from      = Column(String(200), default="")  # откуда груз
    route_to        = Column(String(200), default="")  # куда груз
    distance_km     = Column(Float, default=0.0)
    price_per_km    = Column(Float, default=0.0)
    total_price     = Column(Float, default=0.0)
    
    status          = Column(String(30), default="pending")  
    # pending -> driver_accepted -> in_transit -> delivered -> completed
    # pending -> cancelled_by_buyer
    # pending/driver_accepted -> cancelled_by_driver
    
    buyer_confirmed_disclaimer    = Column(Boolean, default=False)
    driver_confirmed_disclaimer   = Column(Boolean, default=False)
    
    created_at      = Column(DateTime, server_default=func.now())
    updated_at      = Column(DateTime, onupdate=func.now())
    
    order   = relationship("Order")
    courier = relationship("User", foreign_keys=[courier_id])
    buyer   = relationship("User", foreign_keys=[buyer_id])
```

### 1.4 Заказ: новое поле

**`models.py`** — добавить в `Order`:
```python
delivery_request_id = Column(Integer, ForeignKey("delivery_requests.id"), nullable=True)
```

**`main.py`** — миграции:
```python
await safe_exec("ALTER TABLE courier_profiles ADD COLUMN IF NOT EXISTS price_per_km FLOAT DEFAULT 0")
await safe_exec("""
CREATE TABLE IF NOT EXISTS delivery_requests (
    id SERIAL PRIMARY KEY,
    order_id INTEGER REFERENCES orders(id),
    courier_id INTEGER REFERENCES users(id),
    buyer_id INTEGER REFERENCES users(id),
    route_from VARCHAR(200) DEFAULT '',
    route_to VARCHAR(200) DEFAULT '',
    distance_km FLOAT DEFAULT 0,
    price_per_km FLOAT DEFAULT 0,
    total_price FLOAT DEFAULT 0,
    status VARCHAR(30) DEFAULT 'pending',
    buyer_confirmed_disclaimer BOOLEAN DEFAULT FALSE,
    driver_confirmed_disclaimer BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP
)
""")
await safe_exec("ALTER TABLE orders ADD COLUMN IF NOT EXISTS delivery_request_id INTEGER REFERENCES delivery_requests(id)")
```

## ЧАСТЬ 2: Бэкенд — API эндпоинты

### 2.1 Поиск драйверов по маршруту

**`delivery.py`** — новый эндпоинт:
```
GET /api/delivery/couriers/by-route?from=Самарканд&to=Навои
```
Ответ:
```json
{
  "exact": [
    {"id", "user_id", "full_name", "city", "transport_type", "rating", 
     "route_from", "route_to", "price_per_km", "photo_url", "total_deliveries"}
  ],
  "anywhere": [ ... та же структура ... ]
}
```
- `exact` — драйверы с `route_from=X AND route_to=Y`, сортировка по `rating DESC`
- `anywhere` — драйверы с `route_anywhere=True`, сортировка по `rating DESC`

### 2.2 Профиль драйвера (публичный)

Уже есть: `GET /api/courier/public/{user_id}` — расширить, добавить `price_per_km` в ответ.

### 2.3 Калькулятор расстояния

**`delivery.py`** — новый эндпоинт:
```
GET /api/delivery/calculate-route?from=Самарканд&to=Навои&courier_id=5
```
Ответ:
```json
{
  "from": "Самарканд",
  "to": "Навои",
  "distance_km": 270,
  "price_per_km": 100000,
  "total_price": 27000000
}
```
- Используем haversine (уже есть в delivery.py) или упрощённый расчёт по городам
- Для точности: координаты городов хардкодим в словарь ( main города Узбекистана)

### 2.4 Создание delivery request

**`delivery.py`** — новый эндпоинт:
```
POST /api/delivery/request
```
Тело:
```json
{
  "order_id": 123,
  "courier_user_id": 5,
  "route_from": "Самарканд",
  "route_to": "Навои",
  "distance_km": 270,
  "price_per_km": 100000,
  "total_price": 27000000
}
```
- Создаёт `DeliveryRequest` со статусом `pending`
- Привязывает к `Order.delivery_request_id`
- Возвращает `delivery_request_id`

### 2.5 Buyer подтверждает disclaimer

```
PATCH /api/delivery/request/{id}/buyer-confirm
```
- Ставит `buyer_confirmed_disclaimer = True`
- Если `driver_confirmed_disclaimer` тоже True → статус меняется на `driver_accepted`
- Уведомляет драйвера через чатбот

### 2.6 Driver принимает заказ

```
PATCH /api/delivery/request/{id}/driver-accept
```
- Драйвер видит входящий заказ
- Нажимает "Принять" → подтверждает disclaimer
- Ставит `driver_confirmed_disclaimer = True`
- Если `buyer_confirmed_disclaimer` тоже True → статус = `driver_accepted`
- Уведомляет покупателя через чатбот: "Драйвер X принял ваш заказ. Свяжитесь: +998..."

### 2.7 Driver отклоняет / Buyer отменяет

```
PATCH /api/delivery/request/{id}/driver-reject
PATCH /api/delivery/request/{id}/buyer-cancel
```
- Меняют статус, уведомляют другую сторону

### 2.8 Мои заказы (покупатель) — расширить

Существующий `GET /api/orders/my` — добавить в ответ:
- `delivery_request` объект (если есть): `status`, `courier_name`, `courier_phone`, `total_price`, `distance_km`

### 2.9 Заказы драйвера

```
GET /api/delivery/request/my
```
- Возвращает все `DeliveryRequest`, где `courier_id = current_user.id`
- С информацией о заказе (товар, цена, покупатель) и статусом

## Верификация

1. **Фермер** добавляет товар с "Есть доставка" → в каталоге поле видно
2. **Покупатель** на product page → radio "Доставка фермера" видна (товар с доставкой)
3. **Покупатель** выбирает "Внешняя доставка" → модалка → выбирает маршрут → список драйверов (exact → anywhere, по рейтингу)
4. **Покупатель** кликает драйвера → профиль + калькулятор (расстояние * цена_за_км)
5. **Покупатель** нажимает "Выбрать" → disclaimer → оба чекбокса → "Оформить заказ"
6. **Заказ** создаётся в БД, покупатель видит "Ожидание" в "Мои заказы"
7. **Драйвер** получает уведомление → видит входящий заказ → disclaimer → "Принять"
8. **Оба подтвердили** → статус `driver_accepted` → оба получают телефон друг друга через чатбот
