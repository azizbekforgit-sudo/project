# Исправление деплоя Railway

## Проблема

Railway строит из корня `agroverse-fixed/`, но Dockerfile лежит в `agroverse back/`. Railway интерпретирует старт-команду как путь к Dockerfile → крэш.

Ошибка:
```
couldn't locate the dockerfile at path sh -c "uvicorn app.main:app --host 0.0.0.0 --port $PORT" in code archive
```

## Решение: Dockerfile в корне проекта

Создать `Dockerfile` в корне `agroverse-fixed/` (вместо `agroverse back/Dockerfile`):

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY "agroverse back/requirements.txt" .
RUN pip install --no-cache-dir -r requirements.txt
COPY "agroverse back/" .
EXPOSE 8000
CMD ["sh", "-c", "uvicorn app.main:app --host 0.0.0.0 --port ${PORT:-8000}"]
```

Почему не `railway.json`: пробел в имени папки `agroverse back/` может сломать парсер Railway.

## Верификация

1. Railway находит Dockerfile в корне → сборка стартует
2. `pip install` проходит
3. Контейнер стартует → миграция enum→VARCHAR → seed_admin → приложение работает
