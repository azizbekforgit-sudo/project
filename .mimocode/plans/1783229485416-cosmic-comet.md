# Исправление крэша бэкенда AgroVerse

## Корень проблемы

PostgreSQL в продакшене имеет нативный enum-тип `userrole` (значения UPPERCASE: FERMER, XARIDOR, ADMIN, COURIER). Текущие модели определяют `role` как `Column(String(20))`, но колонка в БД до сих пор enum-типа. Когда SQLAlchemy читает User через ORM (`select(User)`), он видит enum-колонку и пытается десериализовать значение `admin` (lowercase) — не находит среди UPPERCASE-значений → `LookupError` → крэш при старте.

Миграция enum→VARCHAR в `lifespan()` пытается это починить, но при ошибке молча проглатывается (`try/except`), и `seed_admin()` падает.

## Что уже исправлено (в предыдущих правках)

1. **`models.py`**: `overlaps=` заменён на `back_populates=` + `viewonly=True` — убирает SAWarning
2. **`main.py`**: Миграция enum→VARCHAR переписана — каждая колонка обрабатывается отдельно + нормализация lowercase значений + удалена битая ссылка на `migrate_sql`
