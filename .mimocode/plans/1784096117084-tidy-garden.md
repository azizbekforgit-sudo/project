# Plan: AgroVerse Chat Module + Decoupled Driver Selection

## Overview
Add a chat system between buyer/farmer/driver roles, tied to orders, with admin moderation. Decouple driver selection from order assignment — selecting a driver becomes "soft" (candidate only), actual assignment happens via "Order this driver" button in chat.

## Architecture Decisions
- **Realtime**: Polling (3s interval) — no WebSocket infrastructure exists, polling is simplest for first iteration
- **File storage**: Local `uploads/chats/` directory, served via existing FastAPI static mount
- **Phone blocking**: Server-side regex before sending message
- **DB migrations**: Add via `ALTER TABLE ADD COLUMN IF NOT EXISTS` pattern (matching existing `main.py` convention)

### 1.2 Add Chat & ChatMessage models
**File**: `agroverse back/app/models.py`
```python
class ChatType(str, enum.Enum):
    BUYER_FARMER = "buyer_farmer"
    BUYER_DRIVER = "buyer_driver"
    DRIVER_FARMER = "driver_farmer"

class Chat(Base):
    __tablename__ = "chats"
    id = Column(Integer, primary_key=True)
    order_id = Column(Integer, ForeignKey("orders.id"), nullable=False)
    type = Column(String(20), nullable=False)  # ChatType
    participant_a_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    participant_b_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    status = Column(String(20), default="active")  # active/closed
    created_at = Column(DateTime, server_default=func.now())
    
    order = relationship("Order")
    participant_a = relationship("User", foreign_keys=[participant_a_id])
    participant_b = relationship("User", foreign_keys=[participant_b_id])
    messages = relationship("ChatMessage", back_populates="chat")

class ChatMessage(Base):
    __tablename__ = "chat_messages"
    id = Column(Integer, primary_key=True)
    chat_id = Column(Integer, ForeignKey("chats.id"), nullable=False)
    sender_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    type = Column(String(20), default="text")  # text/photo/voice/location
    content = Column(Text, nullable=False)
    is_blocked = Column(Boolean, default=False)
    created_at = Column(DateTime, server_default=func.now())
    
    chat = relationship("Chat", back_populates="messages")
    sender = relationship("User")
```

### 2.2 Create chat router
**New file**: `agroverse back/app/routers/chats.py`

Endpoints:
- `POST /api/chats` — create chat (validates participants are part of the order, prevents duplicates)
- `GET /api/chats` — list current user's chats with last message + unread count
- `GET /api/chats/{chat_id}` — get chat details (with participant info)
- `GET /api/chats/{chat_id}/messages` — paginated message history (supports `?before=<id>` for cursor pagination)
- `POST /api/chats/{chat_id}/messages` — send message (validates sender is participant, checks phone number regex, handles photo/voice/location upload)
- `GET /api/chats/{chat_id}/unread` — get unread count (for polling badge updates)

Phone number blocking regex (server-side):
```python
import re
PHONE_PATTERN = re.compile(
    r'(\+?\d{1,3}[\s\-]?)?'           # optional country code
    r'(\(?\d{2,4}\)?[\s\-]?)?'         # optional area code
    r'\d{2,4}[\s\-]?\d{2,4}[\s\-]?\d{2,4}'  # main number
    r'|\d{7,15}'                        # raw digits 7-15 long
)
```

Driver candidate & assignment endpoints (in existing `orders.py` or new router):
- `POST /api/orders/{order_id}/select-driver-candidate` — sets `driver_candidate_id` on order (buyer only)
- `POST /api/orders/{order_id}/assign-driver` — creates DeliveryRequest + assigns driver (replaces automatic flow)

Admin endpoints:
- `GET /api/admin/chats` — list all chats with filters (order_id, user_id, type)
- `GET /api/admin/chats/{chat_id}/messages` — read messages (read-only)
