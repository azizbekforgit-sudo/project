# Plan: Исправление проблем с добавлением продуктов

## Root Cause

У тебя **два конфликтующих бэкенда** на одной PostgreSQL базе:
- `main.py` (корневой) — запускается через Dockerfile, использует UUID-токены, нет колонки `photos`
- `app/main.py` (модульный) — JWT-авторизация, роутеры, колонка `photos`, модерация продуктов

Результат: пользователь регистрируется в одном → авторизуется через другой → 401/403 → продукт создаётся в БД, но API не отдаёт его фронту.

Дополнительно: `api.js` указывает Railway URL `5501`, а CORS в `app/main.py` разрешает `7a95` — другой проект.

**Решение:** Единый бэкенд — модульный `app/main.py` (новее, лучше структурирован).

## Changes

### 1. `agroverse back/Dockerfile` — сменить entrypoint
```
- CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
+ CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 2. `agroverse back/app/main.py` — исправить CORS
Добавить `https://project-production-5501.up.railway.app` в `ALLOWED_ORIGINS`.

### 3. `agroverse front/js/api.js` — синхронизировать URL бэкенда
Убедиться, что `BASE_URL` (строка 5) указывает на тот же Railway-проект, что и CORS. Если бэкенд на `7a95` — обновить фронт. Если на `5501` — обновить CORS.

### 4. Удалить/переименовать корневой `main.py`
`main.py` → `main_legacy.py` чтобы не путал.

### 5. `app/routers/products.py` — проверить создание продукта
Убедиться, что `create_product` (строка 125) корректно сохраняет фото и статус `PENDING`.

### 6. Фронт — показ pending-продуктов
`catalog.js:39` уже обрабатывает `pending` статус (показывает бейдж). Но если пользователь хочет видеть свои pending-продукты — нужно добавить эндпоинт `GET /api/my/products` в модульный бэкенд (его там нет, есть только в корневом).

### 7. `app/routers/products.py` — добавить `GET /api/my/products`
Новый эндпоинт для фермера: список своих продуктов (все статусы).
