# План исправлений: Профиль + Товары + Дизайн драйвера

## Проблемы

1. **404 на `/api/my/products`** — бэкенд на Railway не задеплоен, но `api.js` всё ещё ссылается на этот эндпоинт. Профиль уже переключён на `API.getProducts({})` с клиентской фильтрацией — это работает.
2. **Товары не видны на сайте** — статус `active` уже исправлен в main.py. Но фронтенд не задеплоен.
3. **Дизайн драйвера** — тёмный сайдбар (`#1a2332`) + синие акценты (`#2563eb`) не совпадает с зелёной темой AgroVerse (`#10B981`).

## Что менять

### 1. `agroverse front/js/api.js` — убрать мёртвый эндпоинт
- **Строка 133**: `getMyProducts` ссылается на `/api/my/products` — этот метод больше нигде не вызывается (profile.js уже использует `API.getProducts({})` с фильтрацией). **Удалить строку** `getMyProducts: () => request('GET', '/api/my/products'),` — она не нужна и создаёт путаницу.

### 2. `agroverse front/js/pages/delivery.js` — полный редизайн CSS
Текущий CSS (строки 1886–2324) использует:
- Тёмный сайдбар `#1a2332`
- Синие акценты `#2563eb`
- Базовые тени и скругления

**Нужно переделать на зелёную тему AgroVerse:**
- Сайдбар → `var(--bg-card)` с зелёным градиентом в хедере
- Акценты → `var(--clr-primary)` (#10B981)
- Использовать существующие CSS-переменные из `style.css` (глобальные)
- Скругления `var(--radius)`, тени `var(--shadow)`, шрифты `var(--font-display)`
- Карточки с `var(--bg-card)` и `var(--line)` бордером
- Hover-эффекты с `var(--glow-soft)`
- Кнопки в стиле `.btn-primary` с градиентом

**Конкретные изменения CSS:**
- `.delivery-sidebar` → белый фон, зелёный градиент в `.ds-brand`
- `.ds-nav-item.active` → зелёный вместо синего
- `.hs-card` →обавить зелёные иконки и hover-эффекты
- `.wallet-card` → зелёный градиент вместо синего
- `.ot-tab.active` → зелёный
- `.oc-price` → зелёный
- `.tc-top-badge` → зелёный
- `.pc-fill` → зелёный прогресс-бар
- Фон страницы → `var(--bg)` с атмосферными градиентами

### 3. `agroverse back/main.py` — уже исправлено
- Статус товара фермера: `active` вместо `pending` ✓
- PATCH endpoint для редактирования товаров ✓
- GET `/api/my/products` эндпоинт (для будущего деплоя) ✓
