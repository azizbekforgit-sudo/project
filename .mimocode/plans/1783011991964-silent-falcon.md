## Задача 1: Исправить логин админа

**Проблема:** `config.py` генерирует `secret_key` через `model_post_init` и сохраняет в `.secret_key` файл рядом с `config.py`. Если `.env` не загружается (app запускается не из `agroverse back/`), `admin_phone` и `admin_password` берутся из дефолтов. Но основная проблема — если БД уже содержит старого админа с другим телефоном, `seed_admin()` его не пересоздаст.

**Решение:**
- **Файл:** `agroverse back/app/main.py` — добавить лог: если админ уже есть, вывести его телефон для справки
- **Файл:** `agroverse back/app/config.py` — оставить как есть (уже исправлено)
- **Файл:** `agroverse back/.env` — убедиться что `ADMIN_PHONE=+998000000000` и `ADMIN_PASSWORD=admin123`
- Добавить в `seed_admin()` print что админ уже существует с каким телефоном (для отладки)

**Проверка:** Логин `+998000000000` / пароль `admin123` → редирект на `/admin`

## Задача 2: Зелёный дизайн вместо чёрного

**Текущее состояние:** CSS `:root` уже имеет зелёные токены (`--clr-primary: #0a6e3a`). Но многие компоненты в `style.css` и `home.js` `injectStyles()` используют `var(--ink)` (чёрный `#1a1a1a`) для фонов кнопок, аватаров, бейджей.

**Что нужно исправить:**
- **Файл:** `agroverse front/css/style.css` — проверить все `background: var(--ink)` и заменить на `var(--gradient-primary)` где уместно (аватары, бейджи, иконки)
- Убедиться что `btn-primary` использует зелёный gradient, а не чёрный
- Проверить что навбар, тарифы, AI-отправка — всё зелёное

## Задача 3: Переработка раздела курьеров (Йўлчи)

Это самая большая задача. Разбита на подзадачи.

### 3.1. Новая регистрация курьера с маршрутами

**Текущее состояние:** Курьер заполняет профиль (транспорт, город, радиус) без указания маршрутов.

**Новые требования:**
- При регистрации курьер указывает **маршрут**: откуда и до куда может везти груз
- Кнопка «Из любого места в любое место» — универсальный маршрут
- Дополнительно вводит **адрес** (для будущих функций)
- После выбора маршрута курьер появляется на карте

**Изменения:**

**Бэкенд — `app/models.py`:**
- Добавить в `CourierProfile`: `route_from` (String), `route_to` (String), `address` (String), `is_universal_route` (Boolean, default=False)

**Бэкенд — `app/routers/delivery.py`:**
- Обновить `CourierProfileSetup` schema: добавить `route_from`, `route_to`, `address`, `is_universal_route`
- Обновить `setup_courier_profile` endpoint: сохранять новые поля
- Обновить `couriers_nearby`: фильтровать по маршруту (если курьер не универсальный, искать совпадение `route_from` с адресом клиента)
- Обновить `profile_to_dict`: включить новые поля

**Фронтенд — `js/pages/delivery.js`:**
- Переработать `_renderOnboarding()`: добавить шаг выбора маршрута
  - Шаг 0: Выбор транспорта (сейчас есть)
  - Шаг 1: **Маршрут** — два поля «Откуда» / «Куда» + кнопка «Из любого места»
  - Шаг 2: Адрес, город, радиус, документы
- Сохранять `route_from`, `route_to`, `address`, `is_universal_route` в onboarding state

### 3.2. Новая страница «Найти курьера» (для фермеров/покупателей)

**Текущее состояние:** Простая строка поиска по городу, список курьеров без карты.

**Новые требования:**
1. **Карта** с зоной depot (радиус доставки)
2. Пользователь ставит точку ( Depot ) → определяется радиус
3. Нажимает «Найти курьеров» → показываются курьеры в этой зоне
4. Клик на курьера → открывается **полный профиль**:
   - Все данные из регистрации
   - Маршрут (откуда-куда)
   - Рейтинг (из 10)
   - Фото, транспорт, опыт

**Изменения:**

**Бэкенд — `app/routers/delivery.py`:**
- Обновить `couriers_nearby`: возвращать все поля профиля (route_from, route_to, address, experience_years, bio, photo_url, documents)
- Новый endpoint `GET /api/courier/profile/{user_id}` — публичный профиль курьера (без баланса и приватных данных)

**Фронтенд — `js/pages/delivery.js`:**
- Полностью переписать `_renderFindCourier()`:
  - Hero-секция с заголовком и иллюстрацией
  - **Карта** (Leaflet/OpenStreetMap) — пользователь кликает → ставит точку depot
  - Радиус-слайдер (5-100 км)
  - Кнопка «Найти курьеров»
  - Результаты — карточки курьеров на карте + список
  - Клик на курьера → **модальное окно профиля** с полной информацией

**Фронтенд — `css/style.css`:**
- Новые стили для: карта-контейнер, depot-точка, карточки курьеров на карте, модальное окно профиля курьера

**Фронтенд — `js/api.js`:**
- Добавить `API.getCourierPublicProfile(userId)` — GET `/api/courier/profile/{userId}`

### 3.3. Система рейтинга (из 10)

**Текущее состояние:** `CourierProfile.rating` — float, default 5.0. Админ одобряет/отклоняет курьера без выставления рейтинга.

**Новые требования:**
- Максимальный рейтинг — 10
- При регистрации у всех стоит 0
- После проверки профиля админ выставляет рейтинг (0-10) на основе заполненности профиля

**Изменения:**

**Бэкенд — `app/models.py`:**
- Изменить default `rating` с 5.0 на 0.0

**Бэкенд — `app/routers/delivery.py`:**
- Обновить `approve_courier`: принимать параметр `rating` (int, 0-10)
- Обновить admin schema: добавить `rating` в `CourierProfileSetup` или отдельный endpoint

**Фронтенд — `js/pages/admin.js`:**
- В секции модерации курьеров: добавить input рейтинга (0-10) перед кнопкой «Одобрить»
- Показать заполненность профиля при решении

### 3.4. Визуальные улучшения (кастомные эмодзи, эффекты)

**Текущее состояние:** Секции используют простые эмодзи (🏠📦🗺️💳🤖💰🛒👤), карточки курьеров простые.

**Новые требования:**
- Кастомные эмодзи / SVG иконки, подходящие аграрной тематике
- Вай-фер эффекты: анимации, hover-эффекты, glass-morphism
- Карточки курьеров с градиентами, тенями, анимациями

**Изменения:**

**Фронтенд — `js/pages/delivery.js`:**
- Обновить `TRUCK_TYPES`: кастомные SVG-иконки вместо эмодзи
- Обновить `SECTIONS`: кастомные иконки
- Карточки курьеров: градиентные фоны, hover-эффекты, анимации появления
- Модальное окно профиля: glass-morphism, анимация

**Фронтенд — `css/style.css`:**
- Новые классы: `.courier-card`, `.courier-avatar`, `.courier-profile-modal`, `.depot-map-container`
- Glass-morphism эффекты: `backdrop-filter: blur()`, полупрозрачные фоны
- Hover-анимации: `transform: translateY()`, `box-shadow` переходы
- Анимации появления: `@keyframes` для карточек курьеров
